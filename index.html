Project Objective

The goal of this project is to create a comprehensive dashboard for AdventureWorks that addresses key business needs:

  • KPI Tracking: Monitor essential metrics like revenue, profit, and order volume to identify trends and potential issues quickly.
  • Regional Performance Comparison: Assess performance across different regions to identify strengths and areas for improvement.
  • Product Trends Analysis: Understand product performance to make informed decisions about inventory and marketing.
  • High-Value Customer Identification: Use advanced analytics to profile and target high-value customers effectively.

Implementation Steps

  1. Data Loading: Imported the dataset into Power Query Editor for initial processing.
  2. Data Transformation: Applied necessary transformations, including creating dimension tables like Date and Metrics.
  3. Calculated Tables: Developed Sales Data and Returns Data tables for detailed sales and returns analysis.
  4. Measure Creation: Defined key measures to extract insights from the dataset.
  5. Data Modeling: Established relationships between tables using primary and foreign keys, ensuring correct cardinality.
  6. Dashboard Design: Addressed three key questions to guide the design:
    • What type of data is being handled? (Time series, categorical, geospatial, hierarchical)
    • What is the message to communicate? (Category comparison over time)
    • Who is the audience? (Managers)
  7. Dashboard Pages:
    • Executive Dashboard: Provides an overview of key metrics such as Total Revenue, Total Profit, Orders by Category, etc.
    • Product Detail: Offers detailed insights into product performance, including profit and return rates.
    • Customer Detail: Analyzes customer demographics and purchasing behavior.
  8. Custom Tooltips: Added tooltips to enhance data exploration.
  9. Executive Dashboard Features: Visuals include total revenue, profit, orders, return rate, and monthly revenue.
  10. Product Detail Page: Detailed metrics for selected products using slicers.
  11. Customer Detail Page: Visualizes customer data based on occupation, income level, etc.
  12. Custom Filters: Implemented togglable filters for dynamic data exploration.
  13. Navigation: Added buttons for seamless navigation across different dashboard pages.
  14. Deployment: Uploaded the completed dashboard to Power BI Service.
